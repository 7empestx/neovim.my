# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

Grant's dotfiles repo: Neovim, tmux, and zsh configs. Lives at `~/.config/nvim` with symlinks for zsh and tmux.

## Neovim Load Order

Understanding the load chain is essential:

1. `init.lua` → requires `grant` module
2. `lua/grant/init.lua` → loads in order: `python` → `set` → `packer` → `remap`
3. `after/plugin/*.lua` → plugin-specific configs, loaded **after** packer plugins initialize
4. `ftplugin/*.lua` → filetype-specific overrides (e.g., `markdown.lua` enables word wrap)

## Key Conventions

- **Leader key**: Space
- **Plugin manager**: Packer (`lua/grant/packer.lua`) — run `:PackerSync` after changes
- **Theme**: Tokyo Night Moon (`tokyonight-moon`), with custom line number colors in `after/plugin/colors.lua`
- **LSP**: Mason + mason-lspconfig + lspconfig. Uses `vim.lsp.config` / `vim.lsp.enable` API (not the older `lspconfig.server.setup()` pattern). Currently configured for TypeScript (`ts_ls`).
- **Completion**: nvim-cmp with cmp-nvim-lsp source, LuaSnip for snippets
- **Copilot**: Uses `copilot.lua` (zbirenbaum), NOT `copilot.vim`. Tab to accept, configured in `packer.lua`.
- **Diagnostics**: Virtual text is OFF; uses underline + signs + float (`gl` to open)
- **Indentation**: 2 spaces, no tabs (`set.lua`)
- **Color column**: 80 characters

## When Making Changes

1. **Adding plugins**: Edit `lua/grant/packer.lua`, then `:PackerSync`
2. **Key mappings**: Edit `lua/grant/remap.lua`
3. **Vim options**: Edit `lua/grant/set.lua`
4. **Plugin config**: Add/edit files in `after/plugin/`
5. **Filetype overrides**: Add/edit files in `ftplugin/`
6. **LSP servers**: Add to `after/plugin/lsp.lua` using `vim.lsp.config` + `vim.lsp.enable` pattern
7. **Zsh changes**: Edit `.zshrc`, then `cp .zshrc ~/.zshrc && source ~/.zshrc`
8. **Tmux changes**: Edit `tmux.conf`, then `cp tmux.conf ~/.config/tmux/tmux.conf && tmux source-file ~/.config/tmux/tmux.conf`

## Files Not Tracked

- `plugin/packer_compiled.lua` — auto-generated by Packer
- `~/.secrets` — API keys/tokens, sourced by `.zshrc`
